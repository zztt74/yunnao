# 前端 AI 任务书：真实上手网页问题修复

日期：2026-07-02
分支：`codex/integration`
依据文档：`docs/integration/2026-07-02-hands-on-web-feedback-issues.md`

本文档只分配给前端 AI。前端 AI 负责页面、路由、API client、类型定义、字段映射、错误兜底和前端测试；不得擅自修改后端接口语义、数据库设计或 OpenAPI 主契约。发现后端字段缺失或接口行为不满足时，记录为交接点，由后端 AI 或联调 AI 处理。

## 一、总体目标

修复实际上手网页中暴露出的前端展示、交互、字段映射和错误兜底问题，使患者端、医生端、管理员端在真实后端数据下可稳定演示，不出现白屏、错误字段、假按钮、无意义技术字段或误导性文案。

## 二、任务清单

### F-HW-02：患者端处方页同步展示与白屏兜底

关联问题：HW-02

已定因：
- 患者处方列表在无 encounter 上下文时，`doctorName`、`departmentName`、`patientName` 会被前端映射为空字符串。
- 患者处方页直接调用 `r.doctorName.charAt(0)`，字段为空时可能导致页面渲染异常。

任务：
- 修复患者端处方列表和详情页字段映射，所有展示字段都必须有空值兜底。
- 页面必须覆盖加载中、无处方、401/403、500、字段缺失五类状态。
- 医生确认处方后，患者重新进入处方页必须重新拉取最新数据。
- 如果处方状态仍为 DRAFT，应明确显示不可见或等待医生确认，不要表现成页面故障。

验收：
- 患者端处方页不因 `doctorName`、`departmentName` 等字段缺失白屏。
- 后端返回已确认处方时，患者端列表和详情正常展示。
- 接口失败时显示可读错误和重试入口。

交接：
- 如果患者端接口未返回医生、科室、患者等展示字段，交后端 AI 补响应 DTO。

### F-HW-03：删除管理员首次登录强制改密前端流程

关联问题：HW-03

已定因：
- 后端仍有 `mustChangePassword` 链路，前端不能只隐藏页面后假装完成。

任务：
- 移除管理员首次登录强制跳转改密页逻辑。
- 删除或隐藏首次改密专用入口。
- 保留普通“修改密码/重置密码”能力。
- 若后端仍返回 `mustChangePassword=true`，前端不得强制跳转，但需在交付说明中标记后端残留。

验收：
- 新管理员首次登录后直接进入管理员首页。
- 管理员主动修改密码功能不受影响。

### F-HW-04：管理员用户管理电话和最后登录展示

关联问题：HW-04

已定因：
- 前端当前把 `lastLoginAt` 写死为 `null`。
- 电话列只读账号表字段，医生/患者资料字段缺失时会显示 `--`。

任务：
- 修正用户管理 API 映射，不得用固定空值覆盖真实字段。
- 支持后端新增的 `displayName`、`phone`、`lastLoginAt`、`roles` 等字段。
- 对管理员、医生、患者三类用户分别做兼容展示。
- 最后登录只有后端明确为空时才显示“从未登录”。

验收：
- 有手机号的用户显示真实手机号。
- 登录过的用户显示最后登录时间。
- 三类角色、状态、手机号、最后登录布局稳定。

交接：
- 如果后端仍不返回 `lastLoginAt` 或 profile 手机号，交后端 AI。

### F-HW-05：管理员新增医生表单与错误提示

关联问题：HW-05

已定因：
- 后端密码最小长度为 8 位，前端当前按 6 位校验，导致前端放行后后端返回 400。

任务：
- 将新增医生密码校验和提示同步为后端契约要求。
- 核对 payload：科室提交 `departmentId`，职称、性别、状态提交后端可识别值。
- 新增失败时展示后端返回的具体 `message`，不再只显示 `Request failed with status code 400`。
- 提交按钮禁用状态与表单校验保持一致。

验收：
- 合法医生信息可以新增成功。
- 密码过短、账号重复、手机号不合法、枚举不合法时页面显示明确原因。

### F-HW-06：患者查询改为患者管理

关联问题：HW-06

任务：
- 管理端菜单和页面标题统一改为“患者管理”。
- 页面进入后默认展示全部患者列表，不要求先输入查询条件。
- 保留筛选能力，至少包括姓名/账号/电话关键字、状态。
- 只展示真实可用操作；没有后端接口支撑的操作不要显示为可点击按钮。

验收：
- 管理员进入患者管理页即可看到演示患者。
- 筛选条件可正常缩小列表。
- 页面命名统一为“患者管理”。

### F-HW-07：患者端 AI 分诊多轮上下文

关联问题：HW-07

已定因：
- 前端已具备 `conversationId/history/round` 字段，但仍需确保继续问诊不会重置为独立会话。
- 真实根因在后端多轮实现忽略 history，但前端仍需保证请求链路正确。

任务：
- 继续对话时沿用同一个 `conversationId`。
- 第二轮请求必须携带历史消息、轮次和当前补充内容。
- 请求失败时保留本地对话历史，不清空首轮内容。
- 如果 AI 返回新推荐科室，刷新推荐医生和挂号跳转信息。

验收：
- DevTools Network 中第二轮请求可见 `conversationId`、`history`、`round`。
- 第二轮页面对话记录保留第一轮内容。
- 继续问诊不会重新创建新会话。

交接：
- 如果前端已发送 history 但 AI 仍无上下文，交后端 AI 修 provider prompt。

### F-HW-08：统计驾驶舱前端降级

关联问题：HW-08

已定因：
- 后端设备统计 SQL 会 500，前端 `Promise.all` 会把单接口失败放大成整页失败。

任务：
- 统计页不得因单个统计接口失败整页崩溃。
- 每个统计区域独立降级，显示“暂无统计数据”或可读错误。
- 保留重试入口。
- 若联调 AI 判定演示前无法稳定修复，则按指令临时隐藏菜单，不删除代码。

验收：
- 任意一个统计接口 500 时页面仍可见。
- 演示环境不出现整页白屏或只显示原始 500。

### F-HW-09：登录日志展示修复

关联问题：HW-09

已定因：
- 后端登录审计可能记录 `SYSTEM` 或通用 `USER`，前端又直接展示原始字段。

任务：
- 登录日志表格移除 IP 地址列。
- 用户名优先显示真实账号或姓名，不展示无意义 `SYSTEM` 占位。
- 角色映射为中文：管理员、医生、患者。
- 搜索和筛选项同步移除 IP 相关逻辑。

验收：
- 登录日志无 IP 地址列。
- 管理员、医生、患者登录记录显示正确中文角色。
- 搜索用户名仍可用。

交接：
- 如果后端返回的 username/role 已错误，交后端 AI。

### F-HW-10：操作日志展示修复

关联问题：HW-10

已定因：
- 后端审计字段偏技术化，前端当前直接展示 `action/targetType/targetId`。

任务：
- 操作人显示真实账号/姓名。
- 目标类型翻译为中文业务对象，例如医生、患者、排班、处方、检查检验。
- 目标 ID 作为次要信息展示；若后端提供目标名称或摘要，优先展示目标名称。
- 对空目标、系统操作做友好显示。

验收：
- 操作日志能读出“谁在什么时候对哪个对象做了什么”。
- 内部枚举不直接暴露给演示用户。

### F-HW-11：AI 调用记录供应商和模型展示

关联问题：HW-11

已定因：
- 后端列表 DTO 当前不下发 provider/model，前端把 provider/model 填空。

任务：
- AI 调用记录列表和详情展示供应商、模型字段。
- 真实 DeepSeek 调用显示为 `DeepSeek` / `v4 flash`。
- Mock 调用不得伪装为真实 DeepSeek，应明确显示 Mock 或本地模拟。
- 内部 provider key 不直接展示给用户。

验收：
- 真实 AI 调用记录中供应商显示 `DeepSeek`，模型显示 `v4 flash`。
- Mock 记录与真实记录可区分。

交接：
- 如果后端未返回 display 字段，交后端 AI 补充审计字段。

## 三、前端测试要求

至少补充或更新：
- 患者处方页字段缺失兜底测试。
- 新增医生密码校验和错误提示测试。
- 用户管理字段映射测试。
- 分诊第二轮请求携带 history 测试。
- 登录日志去 IP 和角色映射测试。
- AI 调用记录 provider/model 展示测试。

至少运行：
- `npm run build`
- 前端现有测试集。

## 四、交付说明要求

前端 AI 完成后需输出交付文档，包含：
- 修改文件清单。
- 每个 F-HW 任务完成状态。
- 无法完成项和需要后端/联调处理的证据。
- 构建和测试结果。
