## 2. 输入输出 Schema

### 输入（[PrescriptionReviewAIRequest](file:///d:/shixun/yunnao/backend/src/main/java/com/neusoft/cloudbrain/ai/dto/PrescriptionReviewAIRequest.java)）

| 字段 | 类型 | 说明 |
|---|---|---|
| items | List\<PrescriptionItemInfo\> | 处方药品明细（drugCode、drugName、dosage、frequency、duration） |
| patientAllergies | String | 患者过敏史 |
| deterministicRuleResult | DeterministicRuleResult | **后端确定性规则检查结果（必传）** |

DeterministicRuleResult 子结构：riskLevel、allergyWarnings、interactionWarnings、dosageWarnings、contraindicationWarnings、summary。

不包含患者 ID、姓名、手机号等隐私信息（最小化原则）。

### 输出（[PrescriptionReviewAIResult](file:///d:/shixun/yunnao/backend/src/main/java/com/neusoft/cloudbrain/ai/dto/PrescriptionReviewAIResult.java)）

**Schema 6 字段（对齐 13_AI能力集成AI任务书.md 第3.4节）**：

| 字段 | 类型 | 受控规则 |
|---|---|---|
| riskLevel | String | 枚举 SAFE/LOW/MEDIUM/HIGH/CONTRAINDICATED；不得低于规则命中等级 |
| allergyWarnings | List\<String\> | 过敏风险提示 |
| interactionWarnings | List\<String\> | 药物相互作用提示 |
| dosageWarnings | List\<String\> | 剂量用法提示 |
| recommendations | String | 综合审核建议 |
| summary | String | 审核摘要（含冲突记录，供 AIInvocation 追溯） |

**向后兼容**（禁止修改的 prescription 业务模块依赖旧字段）：
- `suggestions()` 访问器返回 recommendations 值
- `contraindicationWarnings()` 访问器返回空列表（禁忌信息合并到 allergyWarnings / summary）
- `disclaimer()` 方法返回固定安全声明
- 7 参数构造函数兼容旧调用方

## 4. Provider 变化

### AIPrescriptionReviewServiceImpl 改造

[AIPrescriptionReviewServiceImpl](file:///d:/shixun/yunnao/backend/src/main/java/com/neusoft/cloudbrain/ai/application/AIPrescriptionReviewServiceImpl.java) 改造内容：

1. **Schema 校验**：`parsePrescriptionReviewResponse` 校验 3 个必填字段（riskLevel、recommendations、summary），解析 3 个数组字段（allergyWarnings、interactionWarnings、dosageWarnings，缺失时返回空列表）
2. **riskLevel 枚举校验**：仅允许 SAFE/LOW/MEDIUM/HIGH/CONTRAINDICATED，非法值抛 AIInvalidResponseException
3. **风险等级不降级**：`mergeWithRuleResult` 使用 `maxLevel(ruleRiskLevel, aiRiskLevel)` 取规则和 AI 中较高等级
4. **冲突检测与记录**：当 `riskOrder(ruleRiskLevel) > riskOrder(aiRiskLevel)` 时判定为冲突，log.warn 记录，并在 summary 中写入 `[冲突] AI 风险等级 X 低于规则命中等级 Y，已以规则为准` 标记
5. **错误码**：
   - 超时/Provider 异常 → `AI_PRESCRIPTION_REVIEW_FAILED`（HTTP 504）
   - 非法 JSON / Schema 校验失败 / 非法枚举 → `AI_PRESCRIPTION_REVIEW_FAILED`（HTTP 500）

### PrescriptionReviewAIResult DTO 改造

[PrescriptionReviewAIResult](file:///d:/shixun/yunnao/backend/src/main/java/com/neusoft/cloudbrain/ai/dto/PrescriptionReviewAIResult.java) 改造：

- **字段从 7 个调整为 6 个**：移除 contraindicationWarnings、suggestions、disclaimer，新增 recommendations、summary
- **向后兼容**：
  - 7 参数构造函数（旧字段顺序）委托到 6 参数构造函数
  - `suggestions()` 访问器返回 recommendations
  - `contraindicationWarnings()` 访问器返回空列表
  - `disclaimer()` 方法返回固定安全声明
- 兼容禁止修改的 [PrescriptionService](file:///d:/shixun/yunnao/backend/src/main/java/com/neusoft/cloudbrain/prescription/service/PrescriptionService.java#L375) 和 [PrescriptionServiceTest](file:///d:/shixun/yunnao/backend/src/test/java/com/neusoft/cloudbrain/prescription/service/PrescriptionServiceTest.java#L173)

## 6. 风险等级不降级策略

```text
风险等级枚举：SAFE < LOW < MEDIUM < HIGH < CONTRAINDICATED

合并逻辑（mergeWithRuleResult）：
  ruleRiskLevel = 规则命中等级
  aiRiskLevel = AI 输出等级
  mergedRiskLevel = maxLevel(ruleRiskLevel, aiRiskLevel)  // 取较高者

冲突检测：
  if (riskOrder(ruleRiskLevel) > riskOrder(aiRiskLevel)):
      conflict = true
      log.warn("处方审核冲突：AI 风险等级 X 低于规则命中等级 Y，以规则为准")
      summary 追加 "[冲突] AI 风险等级 X 低于规则命中等级 Y，已以规则为准，最终风险等级 Z"
```

- AI 风险等级不得低于规则命中等级（沿用 `maxLevel` 逻辑）
- 冲突时以规则为准
- 冲突记录写入 summary，供 AIInvocation 追溯

## 7. 不得自动确认策略

- PrescriptionReviewAIResult 仅 6 字段，无 approved / status / confirmed 等确认字段
- DTO 从数据结构层面保证 AI 无法产出 approved 状态
- 处方确认由 prescription 业务模块的 PrescriptionService 负责（不在 AI 修改边界内）
- AI 结果仅作为审核建议，最终处方确认必须由医生完成

---

## 8. 超时和降级策略

```text
超时 → 重试 1 次 → 仍失败 → AIInvocation(FAILED) → 抛 AI_PRESCRIPTION_REVIEW_FAILED (HTTP 504) → 业务不影响
非法 JSON → AI_PRESCRIPTION_REVIEW_FAILED (HTTP 500) → 不无限修复 → 降级
缺失必填字段 → AIInvalidResponseException → AI_PRESCRIPTION_REVIEW_FAILED (HTTP 500) → 降级
非法 riskLevel 枚举 → AIInvalidResponseException → AI_PRESCRIPTION_REVIEW_FAILED (HTTP 500) → 降级
```

- 重试仅对超时/5xx，非法 JSON 不重试
- 不无限修复模型输出
- 1 次业务调用 = 1 条 AIInvocation + N 条 AIInvocationAttempt
- AI 审核失败不影响处方创建主流程（由 PrescriptionService 捕获处理，降级为规则风险等级）

## 12. 对后端业务模块的调用说明

- AI 处方审核不直接对外暴露 HTTP，由 prescription 业务模块编排调用
- 调用入口：`AIPrescriptionReviewService.review(PrescriptionReviewAIRequest)`
- 输入必须包含 `deterministicRuleResult`（后端先执行确定性规则检查）
- AI 结果返回 PrescriptionReviewAIResult（6 字段），由 prescription 业务模块写入 PrescriptionReview 实体
- 风险等级不降级由 AI Service 内部 `mergeWithRuleResult` 保证
- 处方确认由 prescription 业务模块负责，AI 不产出 approved 状态

## 13. 潜在安全风险

- Prompt 中明确禁止降级和不得自动确认，但真实模型可能不遵守 → 由 maxLevel 逻辑 + 枚举校验兜底
- 冲突记录写入 summary，真实模型可能不输出冲突标记 → 由 AI Service 内部 mergeWithRuleResult 强制检测和记录
- 输入已做脱敏（不含患者隐私 ID），但需业务模块确保传入前完成脱敏
- 日志和 AIInvocation 中不保存 API Key（由 AIInvocationRecorder 保证）
