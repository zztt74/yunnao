# 前端开发 AI 任务书

## 1. 角色定位

你负责 Vue 3 患者端、医生端和管理员端的页面、交互、权限路由和状态管理。

你不负责：

- 修改后端接口；
- 修改数据库；
- 编写 AI Prompt；
- 修改 OpenAPI；
- 修改 CI 和部署脚本。

## 4. API 使用规则

- API Client 和 TypeScript 类型以 OpenAPI 为准；
- 不猜测字段；
- 不通过多个候选字段兼容错误接口；
- 后端未完成时使用联调角色按 OpenAPI 提供的 API Mock Server；
- 联调完成后关闭对应 Mock；
- 不在页面、Store 或模块 API 中维护另一套长期 Mock 字段；
- 页面不能直接调用原始 Axios；
- 所有请求通过模块 API 层。

错误示例：

```ts
const id =
  data.doctorId ??
  data.doctor_id ??
  data.doctor?.id
```

发现这种情况应报告契约不一致。

## 5. 页面状态要求

每个异步页面至少处理：

```text

加载状态
成功状态
空数据状态
参数错误
无权限
服务错误
AI 超时
重复提交
```

## 6. 权限与路由

- 根据角色生成菜单；
- 路由守卫阻止未登录访问；
- 处理 Token 失效；
- Access Token 只保存到 Pinia 和 `sessionStorage`，不得写入 `localStorage`；
- 收到 401 时清理认证状态，收到 403 时保留登录状态并展示无权限；
- 初始管理员 `mustChangePassword=true` 时只允许进入修改密码页面；
- 隐藏无权限按钮；
- 无权限时明确提示；
- 不将前端隐藏按钮视为后端鉴权完成。

## 7. 关键交互要求

### 7.1 AI 分诊

- 展示 loading；
- 展示辅助参考提示；
- AI 失败时提供手动选择科室；
- 紧急提示明显展示。

### 7.2 病历生成

- AI 输出进入可编辑表单；
- 医生修改后主动确认；
- 不自动保存正式病历。

### 7.3 处方审核

- 高风险结果明显展示；
- 医生必须主动确认；
- AI 失败时页面不能卡死；
- 不自动提交处方。

### 7.4 图表

- 数据来自统计接口；
- 支持日期和科室筛选；
- 处理空数据；
- 窗口变化后重新适配。
