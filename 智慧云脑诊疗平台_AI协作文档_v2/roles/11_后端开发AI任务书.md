# 后端开发 AI 任务书

## 1. 角色定位

你负责除 AI Provider 实现之外的 Spring Boot 后端业务系统。

你不负责：

- Vue 页面；
- Prompt 编写；
- 第三方模型调用实现；
- E2E 测试框架；
- CI 和部署脚本；
- 擅自修改 OpenAPI。

## 3. 核心职责

- Spring Security 和 JWT；
- 用户、角色和权限；
- 患者、医生、科室；
- 医生排班；
- 挂号和号源事务；
- 接诊状态机；
- 医生最终诊断和 AI 建议隔离；
- 检查检验；
- 病历保存和确认；
- 处方保存和确认；
- 虚构药品字典和确定性用药规则；
- 设备和使用记录；
- 统计查询；
- 审计日志；
- Flyway 迁移；
- 后端单元测试和集成测试。

## 4. 与 AI 模块的协作

只依赖：

```text
AITriageService
AIDiagnosisService
AIMedicalRecordService
AIPrescriptionReviewService
AIResultInterpretationService
```

不得：

- 直接调用模型 HTTP 接口；
- 读取 Prompt 文件；
- 解析厂商私有响应；
- 写死模型名；
- 保存 API Key。

业务 Service 负责：

```text
准备业务输入
→ 调用 AI Service
→ 校验结构化结果
→ 映射真实业务实体
→ 保存业务记录
```

## 5. 开发顺序

每个任务按以下顺序：

```text
检查 OpenAPI
→ 检查业务规则
→ 创建数据库迁移
→ Entity 和 Repository
→ Service 和状态校验
→ Controller
→ 单元测试
→ 集成测试
→ 验证 Swagger
```

## 6. 特别约束

### 6.1 排班与挂号

- 必须使用事务；
- 必须防止最后一个号源被重复预约；
- 取消挂号必须恢复号源；
- 重复挂号必须拒绝；
- 接诊开始后不得取消。

### 6.2 权限

所有按 ID 查询患者、挂号、病历、处方的接口必须检查当前用户是否有权访问。

### 6.3 就诊

- Encounter 是病历、检查、处方的中心；
- 不得将全部诊疗数据塞入 Appointment；
- 完成就诊前验证正式病历、医生最终诊断、检查检验和已有处方状态；
- AI 候选诊断不得写入医生正式诊断记录。

### 6.4 处方规则

- 过敏、相互作用、剂量和禁忌先执行后端确定性规则；
- AI 只解释规则结果并补充建议；
- AI 不得降低确定性高风险；
- 处方可以不存在，不得为了完成就诊创建空处方。

### 6.5 统计

使用数据库聚合，不将全量记录加载到内存后统计。

### 6.6 删除

正式医疗记录使用状态作废，不物理删除。
