# 工程开发规范

## 1. 技术栈

### 后端

```text
Java 17
Spring Boot 3
Spring MVC
Spring Security
Spring Data JPA
Validation
springdoc-openapi
Knife4j UI 可选，新增依赖前由后端负责人确认
Flyway
MySQL 8
```

### 前端

```text
Vue 3
TypeScript
Vite
Element Plus
Vue Router
Pinia
Axios
ECharts
```

## 2. 后端分层规范

```text
Controller：参数接收、权限入口、响应
Application Service：用例编排、事务边界
Domain Service：核心业务规则
Repository：持久化
DTO：接口传输
Entity：数据库映射
Mapper：Entity 与 DTO 转换
```

禁止：

- Controller 直接访问 Repository；
- Entity 直接作为 API 响应；
- 跨模块直接调用 Repository；
- 捕获异常后静默忽略；
- 使用散落字符串表示业务状态；
- 将外部 AI 响应直接写入业务表。

## 3. 前端规范

- 使用 Vue 3 Composition API；
- 开启 TypeScript 严格模式；
- API 调用集中在模块 `api/`；
- 页面不得直接创建 Axios 实例；
- 所有异步页面处理 loading、空数据和错误；
- 不通过多字段兼容掩盖契约问题；
- 不在前端计算正式统计口径；
- 表单提交需要防重复；
- 高风险提示需要明确视觉反馈。

## 4. Git 规范

使用短分支：

```text
feature/auth-login
feature/schedule-create
feature/appointment-cancel
feature/triage-consult
fix/appointment-capacity
test/prescription-high-risk
```

不使用长期成员分支：

```text
member-a
member-b
```

提交示例：

```text
feat(schedule): add schedule conflict validation
fix(appointment): restore quota after cancellation
test(triage): add AI timeout fallback
docs(contract): freeze prescription review schema
```

目录和公共文件负责人统一见 `../governance/02_目录所有权.md`。

PR 内容、最低检查和评审流程统一见 `42_PR与交付规范.md`。

## 5. AI 通用禁止事项

- 不重构无关代码；
- 不创建第二套基础设施；
- 不修改其他角色独占目录；
- 不删除测试使构建通过；
- 不伪造测试结果；
- 不硬编码密钥；
- 不使用真实患者数据；
- 不把 Mock 冒充真实模型；
- 不在未说明影响范围时修改接口和数据表；
- 不为兼容错误接口同时读取多个字段名；
- 不跳过权限和归属校验；
- 不将 AI 输出自动确认成正式医疗数据。

## 6. 配置与敏感信息

- 使用环境变量保存数据库密码和 AI Key；
- 使用环境变量保存 JWT 密钥、初始管理员凭据和允许的 CORS Origin；
- 提供 `.env.example`；
- 不提交 `.env`；
- 日志不输出 Token 和密钥；
- 提交前执行敏感信息扫描；
- 演示环境使用虚构账号和虚构患者数据。
- `.env.example` 只能提供变量名和非敏感示例，不提供可直接用于演示环境的真实秘密；
- 缺少必需安全配置时应用应拒绝启动，不使用源码默认密码兜底。

`.env.example` 至少声明：

```text
DB_URL
DB_USERNAME
DB_PASSWORD
JWT_SECRET
INITIAL_ADMIN_USERNAME
INITIAL_ADMIN_PASSWORD
CORS_ALLOWED_ORIGINS
AI_MODE
AI_API_URL
AI_API_KEY
```

`AI_MODE=MOCK` 时 `AI_API_URL` 和 `AI_API_KEY` 可以为空；启用真实 Provider 时必须提供。

## 7. 代码质量

- 关键业务规则必须有测试；
- 公共方法需要明确命名；
- 不使用大而全的 Service；
- 单个方法职责清晰；
- 复杂状态转换集中处理；
- 不允许通过注释保留大量废弃代码；
- 不允许复制粘贴产生重复实现。
