# 质量、测试与完成定义

## 1. 可运行性

系统必须满足：

- 后端可通过 `mvn spring-boot:run` 启动；
- 前端可通过 `npm run dev` 启动；
- 数据库可通过 Flyway 初始化；
- 提供 `.env.example`；
- 不配置真实 AI Key 时可使用 Mock AI；
- 提供启动、初始化和重置说明；
- 前端、后端和数据库版本兼容。

---

## 2. 性能要求

### 2.1 普通业务

- 列表接口必须分页；
- 普通 CRUD 不应明显卡顿；
- 查询避免 N+1；
- 统计优先使用数据库聚合；
- 大列表不一次性返回全部记录。

### 2.2 AI 请求

- 必须有 loading；
- 必须设置超时；
- 重试次数有限；
- 超时后页面可恢复；
- AI 请求不应长期占用业务事务；
- AI 失败不阻断传统流程。

### 2.3 文件上传

仅在已批准启用报告附件等文件功能时适用：

- 限制文件类型；
- 限制文件大小；
- 文件名需要安全处理；
- 大文件不存数据库二进制。

---

## 3. 安全与隐私

- JWT 认证；
- 基于角色的权限；
- 数据归属校验；
- 密码哈希存储；
- 参数校验；
- 防止越权；
- 防止敏感信息进入日志；
- API Key 通过环境变量；
- 不提交 `.env`；
- 演示数据虚构；
- 向 AI 发送的数据最小化；
- 患者敏感信息按需脱敏。
- 密码使用 BCrypt cost 12；
- JWT 密钥长度至少 32 字节并来自环境变量；
- CORS 使用明确 Origin 白名单；
- 登录失败具备限流和锁定；
- 基础版使用 Bearer Token，不使用 Cookie 认证；
- 前端不得使用未经清理的 `v-html` 渲染用户或 AI 内容。

---

## 4. 可靠性

- 数据库写操作使用事务；
- 挂号号源具备并发控制；
- AI Provider 失败可降级；
- 关键状态转换校验；
- 重复提交可识别；
- 异常返回统一结构；
- 正式医疗记录不可随意覆盖；
- 关键操作留审计日志。

---

## 5. 可维护性

- Spring Boot 模块化单体；
- Controller、Service、Repository 分层；
- Vue 按业务模块组织；
- AI 统一 Gateway；
- OpenAPI 为唯一接口契约；
- 数据库使用 Flyway；
- 枚举集中定义；
- 公共组件不重复；
- 目录所有权明确；
- PR 小而可审查。

---

## 6. 可测试性

系统应支持：

- 后端单元测试；
- 后端集成测试；
- 前端类型检查；
- 前端构建；
- 契约测试；
- Mock AI 测试；
- E2E 测试；
- Postman 冒烟测试；
- 数据库重置；
- 固定测试账号和病例。

---

## 7. 可观测性

至少记录：

- 请求 traceId；
- 登录日志；
- 关键业务操作；
- AI 调用耗时和状态；
- 异常堆栈；
- 业务错误码；
- Provider 与模型信息。

不得记录：

- 密码；
- JWT；
- API Key；
- 不必要的患者隐私。

---

## 8. 用户体验

所有异步页面必须处理：

- 初始状态；
- loading；
- 成功；
- 空数据；
- 参数错误；
- 无权限；
- 服务异常；
- AI 超时；
- 重复提交。

特别要求：

- AI 页面显示“仅供辅助参考”；
- 高风险信息明显展示；
- AI 失败提供手动入口；
- 页面不得卡死；
- 图表支持窗口变化；
- 表单错误定位清晰。

## 10. 完成定义

### 10.1 角色子任务完成

角色子任务必须满足：

```text
任务卡范围明确
仅修改授权目录
本角色实现完成
本角色要求的测试通过
接口或依赖与冻结契约一致
交付说明和证据完整
无未报告的跨角色影响
```

角色子任务完成不代表业务功能已经完成。

### 10.2 业务薄切片完成

一个跨角色业务功能必须同时满足：

```text
需求明确
数据库迁移完成
后端接口完成
前端页面完成
权限校验完成
正常流程通过
异常流程通过
OpenAPI 一致
测试通过
无越权修改
文档更新
```

只完成页面或接口不算完成。

纯文档、CI、Prompt 或基础设施任务不强制具备数据库迁移、后端接口和前端页面，但必须满足任务卡中适用于该任务类型的验收项。

## 11. 必测场景

### 11.1 认证和权限

- 正确登录；
- 错误密码；
- 连续失败触发锁定；
- 登录限流；
- 停用账号；
- Token 缺失或过期；
- 退出登录后旧 Token 失效；
- 管理员重置密码后旧 Token 失效；
- 初始管理员首次登录强制修改密码；
- `mustChangePassword=true` 时其他业务接口被拒绝；
- 非白名单 Origin 被 CORS 拒绝；
- 患者访问他人数据；
- 医生访问非本人患者；
- 普通用户访问管理员接口。

### 11.2 排班和挂号

- 排班冲突；
- 号源已满；
- 重复挂号；
- 取消恢复号源；
- 两人竞争最后号源；
- 过期排班；
- 接诊后取消；
- 取消排班同步取消未开始挂号；
- 存在进行中或已完成挂号时拒绝取消排班。

### 11.3 AI 分诊

- 正常返回；
- 高风险；
- 超时；
- 非法 JSON；
- Provider 不可用；
- 不存在科室；
- 手动降级。

### 11.4 接诊和病历

- 非本人医生；
- 重复接诊；
- 待检查；
- AI 病历生成；
- 医生确认；
- 无处方时允许完成就诊；
- 有未完成检查时拒绝完成就诊；
- 缺少医生最终诊断时拒绝完成就诊；
- 基础版本确认后禁止覆盖；启用修订扩展时验证修订留痕；
- AI 失败手工填写。

### 11.5 处方

- 正常审核；
- 高风险；
- 确定性规则命中；
- AI 不得降低确定性高风险；
- AI 失败；
- 医生确认；
- 作废；
- 越权修改。

### 11.6 设备

- 正常占用；
- 重复占用；
- 故障设备；
- 启用维修扩展时验证维修状态；
- 使用结束恢复；
- 状态变化写入历史。

### 11.7 统计

- 门诊量口径；
- 医生接诊量；
- 取消率；
- 设备使用率；
- AI 成功率；
- AI 重试不重复计入业务调用分母；
- 日期和科室筛选；
- `Asia/Shanghai` 自然日边界；
- 历史设备有效管理时长；
- 空数据。

### 11.8 数据库约束

- 用户名、科室编码、药品编码和账号档案关联的唯一约束；
- 一个挂号不能创建多个 Encounter；
- 乐观锁版本冲突被识别；
- 正式医疗记录不能通过 Repository 测试被物理删除。

## 12. PR 最低检查

```text
OpenAPI 校验
后端编译
后端单元和集成测试
前端类型检查
前端构建
契约测试
基础 E2E
目录权限检查
敏感信息扫描
```

任何适用于当前 PR 的步骤失败，不应合并。若某一步不适用，PR 必须写明原因并由对应负责人确认，不得直接省略。

## 13. 最终主流程验收

```text
管理员创建科室和医生
→ 创建排班
→ 患者注册
→ AI 分诊
→ 在线挂号
→ 医生接诊
→ 开立检查
→ 使用设备
→ 录入结果
→ AI 解释结果
→ AI 生成病历
→ 医生确认
→ 创建处方
→ AI 审核
→ 医生确认
→ 完成就诊
→ 患者查看结果
→ 管理员查看统计
```

---

## 14. 最终异常流程验收

至少演示：

### AI 服务失败

```text
AI 超时
→ 结束 loading
→ 提示失败
→ 转人工流程
→ 传统业务继续
```

### 号源已满

```text
提交挂号
→ 后端重新检查
→ 拒绝创建
→ 提示选择其他时间
```

### 越权访问

```text
患者访问他人病历
→ 后端拒绝
→ 返回 PERMISSION_DENIED
```
