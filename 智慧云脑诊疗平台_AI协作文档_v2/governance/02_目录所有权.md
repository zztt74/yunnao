# 目录与公共文件所有权

## 1. 原则

- 每个受治理目录只有一个负责人；
- 角色任务书定义长期所有权，任务卡进一步收窄本次修改范围；
- 任务卡不能扩大角色任务书授权；
- 跨边界修改必须由目录负责人实施，或先获得批准的变更申请；
- 自动检查入口为 `tests/integration/check-ownership.mjs`。

## 2. 目录所有权

| 角色 | 负责目录 |
|---|---|
| 整体架构设计 AI | 协作文档长期规则、`docs/architecture/`、架构规格和计划 |
| 后端开发 AI | `backend/` 非 AI 业务、迁移、`application.yml`、`pom.xml`、非 AI 后端测试 |
| 前端开发 AI | `frontend/` |
| AI 能力集成 AI | 后端 `ai/`、`prompts/`、`application-ai.yml`、AI 测试 |
| 联调测试与集成 AI | `contracts/`、`tests/`、`postman/`、`deploy/`、CI、Compose、集成文档和根公共文件 |

具体正则映射见仓库根目录：

```text
tests/integration/ownership-map.json
```

## 3. 公共文件负责人

| 文件 | 负责人 |
|---|---|
| `backend/pom.xml` | 后端 AI |
| `backend/src/main/resources/application.yml` | 后端 AI |
| `backend/src/main/resources/application-ai.yml` | AI AI |
| `frontend/package.json` | 前端 AI |
| 仓库根 `contracts/openapi.yaml`、公共 Schema、枚举索引 | 联调 AI |
| Prompt 和 AI Provider 配置 | AI AI |
| `docker-compose.yml`、CI、部署脚本 | 联调 AI |
| 根 `README.md`、`.gitignore`、`.env.example` | 联调 AI |
| 协作文档长期规则和任务模板 | 架构 AI |
| 具体任务卡状态和证据 | 当前任务负责人；父任务为联调 AI |
| 根 `docs/changes/approved/` | 联调 AI |

## 4. 两类 Mock

| Mock | 负责人 | 用途 |
|---|---|---|
| `MockAIProvider` | AI AI | 后端内部模拟 AI 能力 |
| OpenAPI API Mock Server | 联调 AI | 后端接口未完成时提供 HTTP 模拟 |

前端不得长期维护另一套 Mock 字段，后端业务模块不得绕过 AI Service 伪造 AI 结果。
