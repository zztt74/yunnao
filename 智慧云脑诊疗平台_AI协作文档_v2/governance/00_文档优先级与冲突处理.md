# 分领域权威来源与冲突处理

## 1. 核心原则

项目不使用一条覆盖所有文档的线性优先级。

不同文档分别回答业务目标、接口格式、架构边界、当前任务和验收方式。发生同领域冲突时按该领域的权威来源判断；发生跨领域冲突时必须停止，不得用某一领域文档直接覆盖另一领域规则。

```text
高层文档决定目标
领域契约决定具体格式
任务卡决定本轮范围
代码只负责实现
跨领域冲突必须停止并走变更流程
```

## 2. 业务语义

决定系统做什么、业务条件和状态如何变化：

```text
已批准的业务或范围变更
→ product/10_项目范围.md
→ product/11_功能需求.md
→ product/12_业务流程与状态机.md
```

适用规则：

- 项目范围决定是否属于基础版本；
- 功能需求描述必须提供的能力；
- 状态机对具体业务转换、前置条件和禁止操作具有更具体的约束；
- 功能需求与状态机不一致时不得自行选择，由架构 AI 和受影响角色确认并提交变更。

## 3. 接口与数据契约

决定 URL、字段、枚举、响应、数据库结构和迁移：

```text
已批准的具体契约变更
→ 仓库根 contracts/openapi.yaml
→ 仓库根 contracts/enums.md
→ 已执行的 Flyway 迁移
→ 后端 DTO、前端类型、Mock 和当前实现
```

适用规则：

- OpenAPI 是 HTTP 接口的唯一机器契约；
- 枚举字典是跨模块枚举索引；
- 已执行 Flyway 代表数据库历史事实，不得直接修改；
- 代码、Mock 和类型必须服从冻结契约；
- OpenAPI 或 Flyway 与业务状态机冲突时停止，不得用技术契约覆盖业务语义。

## 4. 架构、模块和目录边界

决定系统如何拆分、依赖方向以及谁有权修改：

```text
已批准的架构或治理变更
→ architecture/20_整体架构.md
→ architecture/21_模块与依赖边界.md
→ architecture/22_项目目录结构.md
→ governance/02_目录所有权.md
→ 当前角色任务书
→ 当前任务卡的修改边界
```

适用规则：

- 任务卡只能收窄角色任务书授权，不能扩大权限；
- 角色任务书不能突破目录所有权；
- 代码现状不能反向改变架构和目录边界；
- 需要跨边界修改时由目录负责人实施，或先批准变更。

## 5. 当前任务执行范围

决定本轮开发具体做什么：

```text
当前任务关联的已批准变更
→ 当前父任务
→ 当前角色子任务
→ 交付说明和状态变更记录
```

适用规则：

- 父任务定义业务目标、统一契约版本、依赖和总体验收；
- 子任务细化本角色实现，但不能增加父任务未批准的功能；
- 子任务不得改变父任务冻结契约、降低验收标准或扩大目录权限；
- 父任务与子任务冲突时停止，由父任务负责人修正。

## 6. 质量、测试和验收

决定如何证明交付完成：

```text
当前任务验收条件和必须测试
→ workflow/41_质量测试与完成定义.md
→ 契约测试、集成测试和 E2E
→ 本次实际测试结果
```

适用规则：

- 测试通过不代表需求或契约正确；
- 测试与冻结需求或契约冲突时修正测试；
- 任务卡可以增加测试，不得删除适用的项目级最低检查；
- 实际结果是验收证据，不是新的规则来源。

## 7. 代码和非正式信息

以下内容只反映当前实现或沟通，不具有修改需求、契约和权限的效力：

```text
当前代码
→ 代码注释
→ 聊天记录
→ 个人推断
```

代码与已冻结规则冲突时，默认代码需要修正。

## 8. 必须停止的冲突

| 冲突 | 处理 |
|---|---|
| OpenAPI 与业务状态机冲突 | 停止接口开发，提交契约变更 |
| Flyway 与 OpenAPI 冲突 | 停止联调，由后端和联调确认正确契约及迁移方案 |
| 父任务与子任务冲突 | 停止执行，由父任务负责人修正 |
| 任务卡扩大角色权限 | 扩大部分无效，提交架构或所有权变更 |
| 测试与需求或契约冲突 | 修正测试，不迎合错误测试修改规则 |
| 代码与文档或契约冲突 | 默认修正代码 |
| 两份同领域同级文档冲突 | 不自行选择，提交变更申请 |
| 跨领域规则冲突 | 停止，由架构 AI 组织受影响角色评审 |

## 9. 冲突处理流程

```text
停止受影响修改
→ 记录冲突来源和具体条目
→ 说明对角色、接口、数据、测试和部署的影响
→ 将当前任务标记为 BLOCKED
→ 按 governance/03_变更管理.md 提交变更
→ 获得规定审批
→ 更新权威文档和机器契约
→ 解除阻塞并继续任务
```

禁止通过兼容多个字段、保留两套状态、复制基础设施或口头约定规避冲突。
