## 2. 业务目标

为后端、前端、AI 能力和联调四个开发角色建立可独立继续开发的工程骨架、目录所有权和机器可验证契约，不实现具体医疗业务功能。

## 3. 用户角色

```text
主要用户：四个开发 AI 和项目负责人
患者 / 医生 / 管理员：本任务不直接提供业务功能
```

## 4. 前置条件

- AI 协作文档 v2.5 已批准；
- 项目负责人已授权阶段 0 跨角色初始化；
- Java 17、Node.js、Maven 和 Docker CLI 可用；
- Stage 0 不冻结具体医疗业务 URL 和字段。

## 5. 主流程

```text
确认角色和目录所有权
→ 建立 OpenAPI 公共契约
→ 建立后端与 AI 扩展边界
→ 建立前端三类角色路由边界
→ 建立测试、CI 和 Docker 入口
→ 各角色完成自测
→ 联调角色汇总证据
→ 项目负责人评审
```

## 6. 异常流程

```text
契约与共享文档冲突：停止并提交变更申请
需要具体业务 URL 或字段：停止，转入对应业务阶段
需要修改其他角色独占目录：由目录负责人执行或提交变更申请
真实 AI Key 缺失：使用明确标识的 MockAIProvider
Docker 无法启动：保留 Compose 模型验证结果并记录环境阻塞
```
