# 需求或契约变更申请模板

```text
变更编号：
标题：
提出人：
提出时间：
当前状态：DRAFT / REVIEW / APPROVED / REJECTED / IMPLEMENTED
关联任务：
目标版本：
评审人：
```

## 1. 变更原因

说明现有需求、契约或架构为什么无法满足当前业务。

## 2. 当前定义

列出当前文档、OpenAPI、枚举、状态机或数据库定义。

## 3. 建议变更

明确列出：

- URL 和 HTTP 方法；
- 请求与响应字段；
- 枚举或状态转换；
- 错误码；
- 数据库结构；
- 页面与交互；
- Prompt 或 AI Schema。

不适用项写明“不适用”。

## 4. 影响范围

```text
前端：
后端：
AI：
联调与测试：
数据库：
部署：
历史数据：
```

## 5. 兼容与迁移方案

说明是否破坏兼容、是否需要数据迁移，以及旧版本如何退出。

禁止通过同时读取多个候选字段维持长期兼容。

## 6. 测试与回滚

```text
新增或修改测试：
回归范围：
回滚条件：
回滚步骤：
```

## 7. 审批结论

```text
结论：
批准版本：
批准时间：
批准人：
附加条件：
```

只有状态为 `APPROVED` 的变更才能进入 `changes/approved/`，随后由联调角色更新 OpenAPI 和相关契约。
