# CHANGE-003：v2.3 架构治理职责修订

```text
变更编号：CHANGE-003
标题：新增整体架构设计 AI 并明确治理边界
提出人：整体架构复审
提出时间：2026-06-23
当前状态：APPROVED
关联任务：STAGE-0
目标版本：v2.3
评审人：项目负责人
批准人：项目负责人
批准时间：2026-06-23
实施状态：IMPLEMENTED
```

## 1. 变更原因

原文档将协作文档、目录所有权和架构基线交由联调测试与集成 AI 维护，导致执行角色同时维护约束自身的治理规则。任务状态也只有枚举，没有明确转换条件和审批责任。

## 2. 已批准变更

- 新增“整体架构设计 AI”治理角色；
- 协作文档、角色职责、目录结构和任务模板由架构角色维护；
- 联调角色聚焦 OpenAPI、测试、CI、部署和父任务验收；
- 增加任务状态转换、进入条件、审批和退回规则；
- Stage 0 任务类型调整为架构与基础设施；
- 按任务模板补齐 Stage 0 父任务和角色子任务；
- CI 增加目录所有权映射和敏感信息扫描入口；
- 统一 `AI_MODE` 的配置值为大写枚举。

文档信息优先级本次不调整，留待单独评审。

## 3. 影响范围

```text
架构治理：新增独立角色
前端：无业务实现变化
后端：无业务实现变化
AI：AI_MODE 配置值统一
联调与测试：缩小文档治理职责，增加治理检查入口
数据库：无
部署：AI_MODE 示例值调整
```

## 4. 回滚

如需撤销，应新增后续批准变更，不删除本记录。回滚时必须同时恢复角色任务书、目录所有权、任务模板和 CI 治理入口，避免形成不一致状态。
