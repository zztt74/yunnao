## 变更原因

v2 文档存在基础范围与功能要求冲突、状态转换缺失、目录所有权重叠、两类 Mock 混淆和契约治理不足，可能导致四个角色产生不一致实现。

## 已批准变更

- 重新冻结基础版本和扩展功能；
- 补全状态机和统一号源并发策略；
- 分离处方业务状态与 AI 审核状态；
- 修正四角色目录所有权；
- 明确根目录公共文件、`application.yml` 和 `application-ai.yml` 的负责人；
- 明确 `MockAIProvider` 与 API Mock Server；
- 扩展错误码、traceId、时区和 Flyway 规则；
- 增加父任务、角色子任务、变更申请和枚举字典模板；
- 明确阶段 0 的四角色交付物。

## 影响范围

```text
前端：Mock 来源、任务卡、时间和错误处理约束
后端：状态机、并发策略、迁移版本和测试目录
AI：MockAIProvider 边界、Maven 依赖申请方式
联调与测试：文档维护、API Mock、契约和变更记录
数据库：仅调整未来迁移版本规则，未创建迁移
部署：无代码或部署变更
历史数据：无
```

## 回滚

本变更尚未产生项目代码或数据。若团队撤销某项决定，应新增后续已批准变更记录，不直接删除本记录。
