# AI 能力契约规范

## 1. 使用场景

AI 能力集成 AI 在定义 AI Service、Schema、Prompt、Mock、Provider 或降级策略前阅读本文件。

## 2. 边界

```text
业务模块
→ AI Service Interface
→ AI 应用服务
→ AIProvider
→ MockAIProvider / 真实 Provider
```

- 业务模块不感知模型厂商；
- Provider 不直接写业务表；
- AI 输出只生成建议或草稿；
- 医生正式诊断、病历确认和处方确认由业务角色完成；
- 处方确定性规则由后端先执行，AI 不得降低风险等级。

## 3. 能力

| 能力 | 主要输出 | 禁止事项 |
|---|---|---|
| 智能分诊 | 科室编码、优先级、理由和安全提示 | 输出医生 ID 或正式诊断 |
| 辅助诊断 | 候选诊断、证据、缺失信息和检查建议 | 写入医生正式诊断 |
| 病历生成 | 主诉、现病史、既往史和草稿建议 | 编造缺失信息或自动确认 |
| 处方审核 | 风险等级、过敏、相互作用、剂量警告 | 覆盖确定性规则或自动确认 |
| 结果解读 | 异常项、通俗解释和随访建议 | 修改原始检查数值 |

具体字段必须由当前任务卡和对应 Schema 冻结后再实现。

## 4. Mock 场景

业务能力任务至少明确：

```text
正常结果
高风险结果
空结果
超时
非法 JSON
业务引用不存在
Provider 异常
```

Stage 0 只建立 Provider 边界，不冻结具体能力 Schema。

## 5. 结构化输出和记录

- 优先使用 JSON 和 Schema 校验；
- 枚举受控；
- 非法响应映射 `AI_INVALID_RESPONSE`；
- 一次业务调用对应一条 `AIInvocation`；
- 每次 Provider 请求或重试对应一条 `AIInvocationAttempt`；
- 原始响应与解析结果分开记录；
- 日志不保存 API Key；
- 发送给模型的数据最小化；
- 不无限修复或重试。

## 6. Prompt

Prompt 必须：

- 明确任务边界；
- 指定输出 Schema；
- 禁止输出数据库 ID；
- 禁止确定性医疗承诺；
- 明确缺失信息处理；
- 包含“仅供辅助参考”等安全声明；
- 与 Prompt 版本和模型配置一同记录。
