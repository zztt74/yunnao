# 文档版本调整说明

## v2.5

v2.5 将文档优先级改为分领域权威来源：

- 分别定义业务语义、接口数据、架构目录、当前任务和质量验收的权威来源；
- 任务卡只能收窄角色权限；
- 代码和测试结果不能反向修改需求与契约；
- 跨领域冲突必须停止并走变更流程；
- 不再用一条线性顺序强行覆盖不同类型的规则。

## v2.4

v2.4 按子 AI 实际开发场景重新组织文档，不改变业务语义：

- 新增统一开始入口和场景化阅读导航；
- 拆分治理、产品、架构、契约和工作流目录；
- 开发 AI 不再每次通读全部共享文档；
- 角色任务书增加实际任务阅读路径；
- 新增数据库、AI 能力和 PR 专项指南；
- 旧 `shared/` 目录退役为迁移说明；
- 文档优先级内容保持不变，等待单独评审。

## v2.3

v2.3 调整协作治理职责，不改变医疗业务范围：

- 新增整体架构设计 AI；
- 将长期协作文档、角色边界、目录结构和任务模板交由架构角色维护；
- 联调角色聚焦 OpenAPI、测试、CI、部署和父任务验收；
- 增加任务状态转换、审批和退回规则；
- 按模板补齐 Stage 0 任务卡；
- 增加 CI 目录所有权与敏感信息检查入口；
- 统一 `AI_MODE` 配置值；
- 文档优先级保持 v2.2 定义，等待后续单独评审。

## v2.2

v2.2 在 v2.1 基础上补充实施基线：

- 增加 `EncounterDiagnosis`，分离 AI 建议和医生正式诊断；
- 明确无处方也可完成就诊，以及病历、诊断和检查检验完成条件；
- 固定取消排班与未开始挂号的事务联动规则；
- 增加虚构药品成分、相互作用、剂量和禁忌规则；
- 明确确定性用药规则优先于 AI；
- 冻结数据库主键、审计字段、乐观锁、删除、约束、索引和字符集规则；
- 冻结 BCrypt、JWT、Token 失效、CORS、限流和初始管理员安全基线；
- 修正挂号取消率、AI 成功率、设备使用率和自然日统计口径；
- 增加 `DeviceStatusHistory` 支持历史设备有效管理时长；
- 区分 `AIInvocation` 业务调用和 `AIInvocationAttempt` Provider 重试；
- 扩展任务卡中的数据库、安全和确定性规则信息。

## v2.1

v2.1 在 v2 基础上完成一致性修订：

- 统一基础版本、建议功能和扩展功能范围；
- 补全账号、排班、挂号、就诊、病历、处方、检查检验和设备状态规则；
- 明确号源并发使用数据库条件更新；
- 分离处方业务状态与 AI 审核状态；
- 修正后端和 AI 测试目录所有权重叠；
- 补齐根目录公共文件和后端 AI 配置文件所有权；
- 区分后端 `MockAIProvider` 与 OpenAPI API Mock Server；
- 扩展错误码分类、traceId、时间和 Flyway 版本段约定；
- 增加角色子任务与业务薄切片两层完成定义；
- 增加变更申请、已批准变更和枚举字典模板；
- 明确阶段 0 的四角色交付物。

目录名称继续保留 `_v2`，避免仓库链接和已有引用失效；文档内部版本以 v2.5 为准。

本次修订不包含项目代码、OpenAPI 实体契约或具体业务数据库设计。
