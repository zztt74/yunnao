# 开始这里

## 1. 本文件的用途

任何 AI 在分析需求、设计方案或修改文件前，必须先读取本文件。

本文件只回答：

- 我是什么角色；
- 我最少需要读什么；
- 我能否开始任务；
- 什么情况下必须停止；
- 完成后需要交付什么。

具体业务、接口、数据库和测试规则通过 `01_场景化阅读导航.md` 按场景读取，不要求每次通读全部文档。

## 2. 角色

| 角色 | 主要职责 | 不负责 |
|---|---|---|
| 整体架构设计 AI | 协作文档、架构、角色边界、目录结构和治理审核 | 具体业务实现 |
| 后端开发 AI | Spring Boot 业务、数据库、权限、事务和统计接口 | 前端、Prompt、OpenAPI |
| 前端开发 AI | Vue 页面、路由、状态、交互和图表 | 后端、数据库、OpenAPI |
| AI 能力集成 AI | AI Service、Provider、Prompt、Mock、结构化输出和降级 | 传统业务 CRUD、前端、OpenAPI |
| 联调测试与集成 AI | OpenAPI、API Mock Server、契约测试、E2E、CI 和部署 | 业务实现、前端页面、Prompt |

每个 AI 只能修改角色任务书和当前任务卡共同授权的目录。

## 3. 每次任务的最小阅读集

四个开发 AI：

```text
00_开始这里.md
→ 01_场景化阅读导航.md
→ roles/自己的角色任务书
→ 当前父任务
→ 当前角色子任务
→ 当前任务关联的已批准变更
→ 场景导航要求的专项文档
→ 当前任务涉及的代码
```

整体架构设计 AI：

```text
00_开始这里.md
→ 01_场景化阅读导航.md
→ 全部角色任务书
→ 与问题相关的治理、产品、架构和契约文档
→ 当前任务卡和已批准变更
→ 只读取用于验证结构问题的代码
```

禁止无目的扫描整个仓库、其他角色的历史任务卡或无关业务代码。

## 4. 开始任务前检查

同时满足以下条件才能开始修改：

```text
当前角色任务卡存在
任务状态为 READY
负责人角色匹配
目标分支明确
依赖任务已满足
契约版本或提交明确
允许目录和禁止目录明确
验收与必须测试明确
不存在未批准的范围或契约变更
```

不满足时停止并报告，不得自行补充业务规则。

## 5. 标准执行流程

开始前输出：

```text
任务理解
业务目标
涉及模块
拟修改文件
涉及接口和数据表
依赖任务
风险点
计划运行的测试
```

执行过程：

```text
核对任务卡和契约
→ 只修改授权目录
→ 完成本角色实现
→ 运行适用检查
→ 填写交付说明
→ 将任务提交 REVIEW
```

完成后输出：

```text
实际修改文件
实现内容
接口、数据库和配置变化
测试命令与结果
遗留问题
对其他角色的影响
是否可进入联调
```

## 6. 必须停止并报告

- 当前任务卡不存在、不是 `READY`，或负责人角色不匹配；
- 需求、OpenAPI、状态机或任务卡互相冲突；
- 需要修改其他角色独占目录；
- 需要改变已冻结 URL、字段、枚举、错误码或状态转换；
- 当前任务依赖尚未完成；
- 需要创建第二套 Result、JWT、Axios、分页、Mock 或 AI Provider；
- 需要硬编码密码、Token、API Key 或使用真实患者数据；
- 需要删除或降低测试才能通过；
- 实现会突破基础版本范围；
- 当前代码无法编译且原因不属于本角色。

停止后按 `governance/03_变更管理.md` 或父任务阻塞流程处理。

## 7. 通用禁止事项

- 不顺手重构无关模块；
- 不用多套候选字段掩盖契约不一致；
- 不把 Mock 冒充真实能力；
- 不将 AI 输出自动确认为正式医疗数据；
- 不物理删除正式医疗历史记录；
- 不伪造测试或构建结果；
- 不在日志中记录密码、JWT、API Key 或不必要的患者隐私。

## 8. 下一步

根据你正在做的事情，进入 `01_场景化阅读导航.md`。
