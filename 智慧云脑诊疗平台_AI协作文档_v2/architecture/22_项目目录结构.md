# 项目目录结构

## 1. 仓库结构

```text
backend/        后端模块化单体与 AI 扩展边界
frontend/       单一 Vue 应用
contracts/      机器可验证 OpenAPI 和枚举索引
tests/          契约、集成和 E2E
postman/        OpenAPI 派生集合
deploy/         镜像和 Nginx
docs/           项目实施期架构、集成和变更记录
智慧云脑诊疗平台_AI协作文档_v2/  AI 协作治理基线
```

## 2. 后端结构

```text
backend/src/main/java/com/neusoft/cloudbrain/
├── auth/
├── user/
├── patient/
├── doctor/
├── department/
├── schedule/
├── appointment/
├── triage/
├── encounter/
├── examination/
├── laboratory/
├── medicalrecord/
├── prescription/
├── device/
├── statistics/
├── audit/
├── common/
└── ai/
    ├── api/
    ├── application/
    ├── provider/
    ├── dto/
    ├── parser/
    ├── config/
    └── exception/
```

每个业务模块内部按需要使用：

```text
controller/
service/
repository/
entity/
dto/
mapper/
exception/
```

模块之间通过 Service 或公开 API 交互，不跨模块直接访问 Repository。

## 3. 前端结构

```text
frontend/src/
├── api/
├── components/
├── layouts/
├── modules/
├── router/
├── stores/
├── styles/
├── types/
├── utils/
└── views/
```

患者端、医生端和管理端属于同一应用，不建立三套独立工程。

## 4. 协作文档结构

```text
00_开始这里.md
01_场景化阅读导航.md
governance/   协作、任务、所有权和变更
product/      范围、需求和业务流程
architecture/ 系统、模块和目录结构
contracts/    接口、数据库和 AI 契约规范
roles/        长期角色职责
workflow/     开发、测试、PR 和交付
tasks/        当前任务卡
changes/      协作文档基线变更
templates/    可复用模板
```

具体业务实现不得放入协作文档目录。
